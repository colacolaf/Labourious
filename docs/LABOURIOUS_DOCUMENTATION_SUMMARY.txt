QUICK REFERENCE: THE 30-SECOND UNDERSTANDING
================================================================================
 
Labourious = Local AI trading warroom with encrypted credentials and retro UI
 
What it does:
  - Users download and run on their own machine (no cloud)
  - Connect broker accounts securely
  - Create AI agents that follow user-written trading rules
  - Allocate capital across 3 trading strategies (day, swing, long-term)
  - Monitor agents in a retro 2D isometric "warroom" interface
  - Agents can execute trades autonomously or wait for user approval
 
Why it's different:
  - NO cloud sync of API keys (encrypted locally only)
  - NO mandatory cloud LLM (uses free local Ollama by default)
  - NO templated agents (users write their own rules in context files)
  - NO complexity (simple, retro interface)
  - NO unnecessary features (focused on core trading value)
 
Who it's for:
  - Retail investors who want to experiment safely
  - Active traders who value privacy
  - Developers who want to extend the system

THE CORE CONCEPT: CONTEXT FILES
================================================================================
 
Everything in Labourious revolves around context files.
 
A context file is a set of trading rules in plain English (or JSON) that tells an agent how to trade.
 
Example context file (plain English):
  AGENT NAME: Tech Value Investor
  ROOM: Long-term
  ASSETS: AAPL, MSFT, NVDA
  
  BUY RULES:
  - P/E ratio < 15
  - Price/Book < 2
  - Dividend yield > 3%
  
  SELL RULES:
  - P/E ratio > 25
  - Take profit at +50%
  - Stop loss at -15%
 
When the agent runs:
  1. Fetches live market data (prices, indicators)
  2. Reads the context file
  3. Asks LLM: "Based on these rules and current data, should we buy/sell/hold?"
  4. LLM responds with decision + confidence
  5. Agent executes or asks user for approval (depending on mode)
 
This makes agents explainable and editable by non-programmers.

THE ARCHITECTURE IN ONE DIAGRAM
================================================================================
 
User Interface (Web Browser or Electron App)
         ↓ HTTP/WebSocket
Backend Server (Python FastAPI)
         ├─ Agent Orchestrator (runs agents on schedule)
         ├─ LLM Router (sends decisions to Ollama or Claude API)
         ├─ Trade Executor (places orders with brokers)
         ├─ Encrypted Vault (stores API keys safely)
         └─ SQLite Database (logs trades and performance)
         ↓
External Services
         ├─ Ollama (local LLM, free, offline)
         ├─ Broker APIs (Kraken, Interactive Brokers, Coinbase)
         └─ Optional: Claude or GPT APIs (for better reasoning)
 
Key: Everything runs locally. User controls encryption keys. No cloud dependency.

THE WARROOM INTERFACE
================================================================================
 
What you see:
  - A 2D isometric "stadium" view (like retro football game)
  - Agents positioned as colored cards/sprites on the floor
  - Three sections: Day Trading Room, Swing Trading Room, Long-Term Room
  - Click an agent to open inspector panel

  - Trades: Recent trades with entry/exit prices and reasons
  - Rules: Your context file displayed
  - Performance: Sharpe ratio, max drawdown, daily returns
  - Settings: Execution mode, capital allocation, broker connection
 
Interactions:
  - Click agent to inspect
  - Slider to adjust capital allocation (e.g., 60% long-term, 30% swing, 10% day)
  - Toggle button to enable/disable agent
  - Approve/Reject buttons for trades needing user approval
  - Upload new context file to update agent rules

SECURITY MODEL (SIMPLE VERSION)
================================================================================
 
User Sets Vault Password → Password encrypted with PBKDF2 → Derives key → 
Key used to encrypt API keys with AES-256 → Keys stored in ~/.labourious/vault.db
 
Result:
  - Keys never leave user's machine
  - Keys encrypted at rest
  - Keys encrypted in transit (HTTPS to brokers)
  - Vault password NOT stored (can't be recovered, so user must remember)
  - Encrypted backup file created for recovery (user-controlled)
 
If system is compromised:
  - Attacker gets encrypted blob (useless without vault password)
  - User can change vault password and rotate API keys
 
Best practices:
  - Use strong vault password (8+ characters, mixed case, numbers, symbols)
  - Back up encrypted keyfiles regularly
  - Create API keys with limited permissions (read-only for testing)
  - Rotate API keys every 6 months
 
================================================================================
EXECUTION MODES EXPLAINED
================================================================================
 
Each agent can operate in one of three modes:
 
1. AUTONOMOUS
   Agent makes decision → immediately executes trade
   Best for: Long-term room (low frequency, low risk)
   Warning: Executes 24/7 without human intervention
 
2. HUMAN-IN-LOOP
   Agent makes decision → sends notification to user
   User has 30 seconds to approve/reject
   If no response → trade is rejected (safe default)
   Best for: Swing trading and day trading
 
3. RISK-BASED
   Agent evaluates risk level of trade
   Low-risk trades (small size, high confidence) → auto-execute
   High-risk trades (large size, low confidence) → wait for approval
   Best for: Hybrid approach
 
You configure the mode per agent in settings.

================================================================================
PAPER TRADING (REQUIRED BEFORE LIVE)
================================================================================
 
Paper trading simulates trades without real money:
 
What it does:
  - Agent makes trading decisions as if live
  - Uses real market data
  - Simulates execution (with configurable slippage and commission)
  - Shows what would happen without risking capital
 
Best practice:
  - Start all agents in paper mode
  - Run for 2-4 weeks
  - Review trades weekly
  - If performance looks good, switch to LIVE mode
  - Start LIVE with 1-5% of capital
  - Scale up gradually as confidence builds
 
Red flags (don't go live if you see these):
  - Win rate < 40% (worse than flipping a coin)
  - Max drawdown > 30% (too risky)
  - Sharpe ratio < 0.5 (poor risk-adjusted returns)
  - Only a few trades (not enough data)

COMMON MISTAKES & HOW TO AVOID THEM
================================================================================
 
Mistake 1: Writing rules that are too strict
  Problem: Agent never trades (no signals match)
  Solution: Use OR instead of AND, relax thresholds
 
Mistake 2: Writing rules that are too loose
  Problem: Agent trades constantly, loses money
  Solution: Add confirmation signals, tighten thresholds
 
Mistake 3: Curve fitting
  Problem: Rules work great on past data, terrible on live data
  Solution: Use round numbers (RSI < 30, not 32.7), test on multiple timeframes
 
Mistake 4: Ignoring risk
  Problem: Position gets too large, drawdown exceeds comfort level
  Solution: Set max position size (% of account), set max drawdown limit
 
Mistake 5: Not monitoring early
  Problem: Agent runs silently, user unaware of issues
  Solution: Check on agents daily for first week, then weekly
 
Mistake 6: Going live without paper trading
  Problem: Lose money learning what doesn't work
  Solution: Mandatory 2-4 week paper trading period
 
See AGENT_CREATION.md for more details.

EXAMPLE: YOUR FIRST AGENT (30 MINUTES)
================================================================================
 
Step 1: Define Your Edge (5 min)
  "I notice Bitcoin often overshoots and reverses. I want to buy when it's oversold
   (RSI < 30) and sell when it's overbought (RSI > 70). Hold for 2-5 days."
 
Step 2: Write Context File (5 min)
  Save as contexts/btc_momentum.txt:
  
  AGENT NAME: BTC Momentum Trader
  ROOM: Swing Trading
  ASSET: BTC/USD
  BROKER: Kraken
  
  POSITION SIZE: $2,000 max
  MAX POSITIONS: 1
  HOLDING PERIOD: 2-5 days
  
  BUY RULE:
  - RSI(14) < 30 (oversold)
  - Volume > 20-day average
  
  SELL RULE:
  - RSI(14) > 70 (overbought), OR
  - Position up 15% (take profit), OR
  - Position down 5% (stop loss)
 
Step 3: Create Agent Config (5 min)
  Save as agents/btc_momentum.json:
  
  {
    "name": "BTC Momentum",
    "enabled": true,
    "room": "swing_trading",
    "context_file": "contexts/btc_momentum.txt",
    "asset": "BTC/USD",
    "broker": "kraken",
    "execution_mode": "human_in_loop",
    "allocation_percent": 15,
    "max_position_size": 2000,
    "check_frequency": "daily"
  }
 
Step 4: Upload to Labourious (2 min)
  - Open app
  - Go to Swing Trading room
  - Click "Add Agent"
  - Upload btc_momentum.json
  - Set to PAPER TRADING mode
 
Step 5: Test & Monitor (ongoing)
  - Watch trades for 2-4 weeks
  - Review rules if performance is poor
  - Once confident, switch to LIVE mode
 
Done! You now have an agent trading for you.
